## Quality standards

### Code examples
- Complete, runnable code
- Proper error handling
- Realistic data
- Expected outputs
- Tested thoroughly
- Language and filename specified
- Explanatory comments for complex logic
- No real API keys

### API documentation
- All parameters documented
- Success and error examples
- Rate limiting information
- Authentication examples
- HTTP status codes
- Complete request/response cycles

### Accessibility
- Descriptive alt text
- Actionable link text
- Proper heading hierarchy (H2+)
- Keyboard navigation
- Color contrast
- Scannable structure

## Component selection

- **Steps**: Procedures, sequential instructions
- **Tabs**: Platform-specific, alternatives
- **CodeGroup**: Same concept, multiple languages
- **Accordions**: Progressive disclosure
- **RequestExample/ResponseExample**: API endpoints
- **ParamField**: API parameters
- **ResponseField**: API responses
- **Expandable**: Nested objects, hierarchies
